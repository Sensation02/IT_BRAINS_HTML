Head

head - це конфігураційний тег. В ньому задаються параметри нашого сайту. Далі буде розповідатися про все, що пов'язано із цим тегом.

title - тег назви сторінки сайту у відкритих вкладках браузера.

base - тег тег який вказує початковий URL шлях до файлів (тобто до файлу на сервері).

    Атрибути тегу:

  1. target="_blank" - відкривається у новому вікні
  2. target="_top" - відкривається у батьківському вікні
  3. target="_parent" - у батьківському фреймі
  4. target="_self" - в тому ж фреймі

meta - тег який вказує метадані документа, дуже потрібно для СЕО.

    Атрибути тегу:

  1. charset="UTF-8" - шифрування
  2. name="description" content="Це тестова сторінка" - name - вказуємо тип метаданих; content - контент для назв метаданих; description - опис сторінки.
  3. name="generator" content="HTML" - generator - яка технологія використовується для генерації сторінки сайту; не є обов'язковим.
  4. name="keywords" content="HTML, CSS, JavaScript" - keywords - список ключових слів.
  5. name="author" content="Bazzil aka Sensation02" - author - хто створив.
  6. name="viewport" - вказує як треба зображати контент
      content=
    • viewport-fit=cover
    • width=device-width
    • initial-scale=1.0
    • maximum-scale=1.0
    • user-scalable=no

    Всі ці параметри треба вказувати в лапках.
Основні будівельні блоки:
header - Тег вступної секції, шапка. Може бути декілька, в залежності від потреб.

span - Тег-контейнер для тексту, який його може стилізувати.

р - Параграф. Тег тексту незалежної секції.

main. Вміст цього елемента має бути унікальним для документа. Такого тегу не має бути більше одного. Він не має походити від article, aside, footer, header, nav, тобто тегів які не мають основного контенту.

article - Тег який визначає контент статті в документі.

в section знаходиться текст незалежної секції сайту

nav - тег який визначає навігаційні посилання. Далі йдуть приклади навігаційного посилання. Тег посилання - а. Зазвичай навігаційна панель зверху, під шапкою сайту бо вище.

  • HTML
  • CSS
  • JS

aside - тег бічної панелі, яка знаходиться справа або зліва.

  • HTML
  • CSS
  • JS
footer - Тег який визначає кінцеву секцію, або підвал
Глобальні атрибути
  • hidden - сховати тег з екрану
  • id='value' - надати ідентифікатор тегу
  • class='value' - надати класу тегу
  • title='value' - надати заголовок тегу
  • style=''value - надати стилю
  • lang='value' - інформація про мову контенту
  • contenteditable - вказує, якщо треба дати змогу редагувати вміст тегу
  • dir='ltr' або 'rtl' - вказує напрямок тексту, зліва-направо або справа-наліво.
  • draggable='true' - змога переміщати контент мишкою

em - Тег який наголошує, написаний курсивом, ніби з певним наголосом.

strong - тег дуже важливого тексту

b - bold, просто жирний текст, але семантично цей текст не важливий

mark - тег який, ніби маркером, щось позначає

small - Тег який виділяє неважливий, маленький текст

i - тег який виділяє текст в альтернативному голосі, для визначень , назв чогось, щось що в "", цікаве, дивне тощо 

          Pre - тег який не потрібно форматувати, 
          тобто виведе текст так як був написаний 
          в коді

s - тег який позначає текст, який став неправильним, не точним, не актуальним, або застарілим з точки зору контенту

del - тег який позначає текст який видалений. Він як s, але для видаленого тексту)

u - тег який визначає контент, який має помилки.

ins - тег який визначає текст який був доданий.

code - тег який визначає комп'ютерний код

sub - H2O - тег який визначає підрядковий текст

sup - 42 - тег який визначає надрядковий текст

cite - тег який вказує назву певного контенту. Наприклад книга, фільм,картина. Такий контент відображається курсивом.

kbd - тег який вказує що контент є комбінацією клавіш. ctrl+c

samp - тег який вказує, що контент є інформацією від, наприклад, терміналу.

var - тег який вказує, що контент є змінною в програмуванні або математичному виразі. такий контент буде відображатися курсивом.

address - тег який визначає
контактну інформацію про автора
статті, сайту тощо. Також курсивним шрифтом.

q - Тег який вказує, що контент є цитатою. Такий контент матиме лапки по боках.

blockquote - Тег який визначає великий об'єм контенту який був процитований з іншого HTML документа. атрибут cite="URL" - посилання на HTML звідки взята цитата.

abbr - тег який вказує що контент є абревіатурою. HTML, CSS, JavaScript. Такому контенту надається пунктирне підкреслення. атрибут title="value" - пояснення до контенту при наведенні на нього.

dfn - тег який вказує що контент є терміном, який вперше з'явився в HTML документі. такий термін відображається курсивним шрифтом. title="value" - пояснення до контенту.

Теги які не мають повної підтримки:

Тобто не всі браузери не мають їх підтримку. Старі браузери можуть не підтримувати. Або нові браузери які не підтримують старі теги.

time - Open from 10:00 to 21:00

data - тег який перекладає контент на мову для програм, де value="" вказує на якусь частину з бек-енду.

Списки

Впорядкований список: ol, li - list item

reversed - зворотня нумерація списку. start="number" - з якого номера починати. type - 1/A/a/I/i - тип впорядкованого списку. value - атрибут li - вказує номер елемента списку, працює лише з ol.

start="A" type="A":

  1. Tea
  2. Coffee
  3. Milk

type="1":

  1. Tea
  2. Coffee
  3. Milk

type="I":

  1. Tea
  2. Coffee
  3. Milk

type="i":

  1. Tea
  2. Coffee
  3. Milk

reversed type="i":

  1. Tea
  2. Coffee
  3. Milk

ul - unordered list:

  • Tea
  • Coffee
  • Milk
Списки та скрипти
dl (definition list), dt (definition title), dd (definition description)
"dl"
тег контейнер для списку описів термінів.

"dt"
тег який обгортає контент назви термінів.

"dd"
тег який обгортає пояснення терміна.

hr - тег який відокремлює різний тематичний контент. Він надає горизонтальну лінію, яка використовується для розділення незалежного вмісту в HTML документі.

details - тег-контейнер для контенту, який є додатковим деталями, які користувач може відкривати та закривати. атрибут "open" вказує, що контент має бути зразу відкритим.

summary - тег для заголовка деталей. При натисканні на нього буде відкриватися контент в details

dialog - тег-контейнер для діалогового вікна. Такий контент буде відображатися по центру вікна. Атрибут "open" вказує, що діалоговий елемент активний і користувач може його бачити та з ним взаємодіяти
