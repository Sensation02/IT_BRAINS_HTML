Head

head - це конфігураційний тег. В ньому задаються параметри нашого сайту. Далі буде розповідатися про все, що пов'язано із цим тегом.

title - тег назви сторінки сайту у відкритих вкладках браузера.

base - тег тег який вказує початковий URL шлях до файлів (тобто до файлу на сервері).

    Атрибути тегу:

  1. target="_blank" - відкривається у новому вікні
  2. target="_top" - відкривається у батьківському вікні
  3. target="_parent" - у батьківському фреймі
  4. target="_self" - в тому ж фреймі

meta - тег який вказує метадані документа, дуже потрібно для СЕО.

    Атрибути тегу:

  1. charset="UTF-8" - шифрування
  2. name="description" content="Це тестова сторінка" - name - вказуємо тип метаданих; content - контент для назв метаданих; description - опис сторінки.
  3. name="generator" content="HTML" - generator - яка технологія використовується для генерації сторінки сайту; не є обов'язковим.
  4. name="keywords" content="HTML, CSS, JavaScript" - keywords - список ключових слів.
  5. name="author" content="Bazzil aka Sensation02" - author - хто створив.
  6. name="viewport" - вказує як треба зображати контент
      content=
    • viewport-fit=cover
    • width=device-width
    • initial-scale=1.0
    • maximum-scale=1.0
    • user-scalable=no

    Всі ці параметри треба вказувати в лапках.
Основні будівельні блоки
header - Тег вступної секції, шапка. Може бути декілька, в залежності від потреб.

span - Тег-контейнер для тексту, який його може стилізувати.

р - Параграф. Тег тексту незалежної секції.

main. Вміст цього елемента має бути унікальним для документа. Такого тегу не має бути більше одного. Він не має походити від article, aside, footer, header, nav, тобто тегів які не мають основного контенту.

article - Тег який визначає контент статті в документі.

в section знаходиться текст незалежної секції сайту

nav - тег який визначає навігаційні посилання. Далі йдуть приклади навігаційного посилання. Тег посилання - а. Зазвичай навігаційна панель зверху, під шапкою сайту бо вище.

  • HTML
  • CSS
  • JS

aside - тег бічної панелі, яка знаходиться справа або зліва.

  • HTML
  • CSS
  • JS
footer - Тег який визначає кінцеву секцію, або підвал
Глобальні атрибути
  • hidden - сховати тег з екрану
  • id='value' - надати ідентифікатор тегу
  • class='value' - надати класу тегу
  • title='value' - надати заголовок тегу
  • style=''value - надати стилю
  • lang='value' - інформація про мову контенту
  • contenteditable - вказує, якщо треба дати змогу редагувати вміст тегу
  • dir='ltr' або 'rtl' - вказує напрямок тексту, зліва-направо або справа-наліво.
  • draggable='true' - змога переміщати контент мишкою

dfn - тег який вказує що контент є терміном, який вперше з'явився в HTML документі. такий термін відображається курсивним шрифтом. title="value" - пояснення до контенту.

Гіпертекст та ресурси

a - тег для гіперпосилання в HTML документі.

    Атрибути

  • href - вказує URL шлях до файлу
    • Абослютний - повний шлях до файлу
    • Відносний - шлях відносно поточного розташування
    • Якірний - до певного тегу в документі
    • Скриптовий - вбудовує ДЖС код
    • Протокольний - до певного файлу в іншому протоколі
  • target - як відкрити посилання.
  • type - вказує MIME-тип файлу за посиланням.
  • media - приймає будь який дійсний медіазапит.
  • hreflang - визначає мову.
  • download - вказує, що посилання це файл і його треба завантажити з певно назвою.
  • rel - відношення до поточного посилання

source - тег який вбудовує ресурси в HTMl документ. Ресурс - це сторонній файл: фото, аудіо або відео.

    Атрибути

  • media - приймає будь який дійсний медіазапит
  • srcset - вказує посилання для картинки
  • src - вказує посилання для аудіо/відео файлу
  • type

picture - тег контейнер для тегу source, який дає можливість вказати фото ресурси.

audio - тег-контейнер для тегу source, який дає можливість вказати аудіо ресурси.

    Атрибути

  • autoplay - аудіо почне грати як тільки завантажиться.
  • controls - вказує, що аудіоелементи повинні відображатися.
  • loop - вказує, що аудіо почнетсья заново, щоразу, коли воно закінчиться.
  • muted - аудіо має бути вимкнутим.
  • src - шлях до аудіо.
  • preload - вказує як завантажити файл
    • metadata - браузер завантажує лише метадані під час завантаження сторінки.
    • auto - завантажує весь аудіо файл.
    • none - Не завантажує файл під час завантаження сторінки.

track - тег який визначає тектсові доріжки (субтитри) для елементів audio/video.

    Атрибути

  • default - вказує, що трек повинен бути ввімкненим
  • label - вказує заголовок текстового треку
  • src - шлях до файлу
  • srclang - мова
  • kind - вказує тип текстової доріжки
    • captions - підписи
    • chapters - глави
    • descriptions - описи
    • metadata - метадані
    • subtitles - субтитри

video - тег-контейнер для тегу source, який дає можливість вказати відео ресурси.

    • metadata
